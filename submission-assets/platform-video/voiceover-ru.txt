Начинаем с Workflow. Это не просто верхняя hero-страница, а полная карта продукта: molecular input проходит через reference-hit evidence, hard gates, safe-rank decision, publication readiness, Evidence Pack, Evidence Map and Evidence Graph. При прокрутке видно, как последовательность превращается в evidence-bounded claim, а не в неограниченный species truth.

Overview показывает конкурсную подачу и теперь содержит явную ссылку на source repository: Ecooddworld slash EcoGenesis Evidence Atlas. Главный claim остаётся позитивным: EcoGenesis converts barcode, metabarcoding, Sequence ID and BLAST-like outputs into safe taxonomic rank decisions, GBIF publication readiness, repair actions and Evidence Packs.

Run Compiler — практический экран. Здесь видны choose input, validate input, run gates and export pack. Пользователь может начать с mixed demo, CSV upload или reference FASTA search. После запуска результат показывает processed records, species-safe, genus-safe, weak, review-only, not-publishable rows, blockers and repair actions.

Evidence Map связывает GBIF occurrence context, molecular claim states, blockers and export boundaries. Это spatial evidence view, а не claim-промоутер. Карта и summary cards показывают snapshot hash, VSEA rows, safe claims, weak hypotheses and blocked export boundaries, while claim strength stays governed by molecular and publication gates.

Evidence Graph делает provenance видимым. Source, run, snapshot, segments, taxa, claims and edges находятся в одном audit view. Важное ограничение сохраняется: visualization cannot upgrade weak, blocked or review-only claims. Граф объясняет evidence structure, but does not change claim_state.

Exports в Evidence Map показывают, что Observatory создаёт не только визуализацию, а проверяемый package: gbif export preview, ai-ready dataset, source registry audit, graph, VSEA, verification report and ZIP. AI-ready outputs preserve graph states and cannot turn weak hypotheses into positive verified rows.

Judge view собирает release evidence в один экран. Здесь видны contest readiness, verification files, competition reports and adversarial reports. Это важно для конкурса: судья может открыть not only the UI, but the generated reports, ZIP entries, graph audits, VSEA parquet and guardrail checks.

Fragment Graph проверяет marker-level evidence. Пользователь выбирает reference dataset, sequence ID и DNA marker fragment, затем строит taxon graph. Результат показывает hit comparison, safe LCA, lineage path and source monitor. Shared or ambiguous fragments stay genus-safe, higher-rank-safe or review-only instead of becoming species-specific claims.

Validation показывает, как проект проверяется batch-wise. Competition batch, adversarial stress run and Evidence Map verification separate expected decisions from observed decisions. Здесь важны hard-gate failures, graph roundtrip, VSEA parquet, export guardrails and source provenance, not marketing language.

Methods and Audits — место для научных границ и формальной логики. При прокрутке видно input model, match gates, competitor and LCA downgrade, barcode gap, diagnostic k-mer gate, marker profile, assay evidence and publication readiness. Species-level export allowed only when all required gates pass.

Evidence Pack — финальный продуктовый артефакт. Пользователь получает CSV, JSON, HTML, VSEA, graph and audit files: sequence safety table, publication blockers, Darwin Core templates, DNA-derived extension templates, hard gate audit, math viability audit and provenance reports. Platform direction is clear: ingestion adapters, reference registry, repair optimizer, Evidence Map dashboard, graph provenance and AI export guardrails.
