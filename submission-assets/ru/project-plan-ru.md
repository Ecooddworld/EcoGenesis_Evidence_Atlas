# EcoGenesis Evidence Atlas: полный русский план проекта

## 1. Самая простая формулировка

EcoGenesis Evidence Atlas — это инструмент, который берет открытые данные GBIF о находках видов и превращает их в понятный “паспорт доказательств”.

Обычный пользователь видит не просто точки на карте, а ответ:

- что по этим данным можно уверенно сказать;
- что можно сказать только с осторожностью;
- что говорить нельзя;
- какие данные надо процитировать;
- где есть пробелы наблюдений;
- какие ошибки стоит отправить владельцам наборов данных.

Главная идея: GBIF уже помогает находить биологические данные. EcoGenesis помогает правильно ими пользоваться.

## 2. Почему это вообще нужно

Многие люди видят карту с точками и думают: “если точки есть — вид там есть, если точек нет — вида там нет”. Это опасная ошибка.

В реальности отсутствие точки часто означает не отсутствие вида, а отсутствие наблюдений, плохую оцифровку, старые данные, крупную неопределенность координат или недостаток источников.

Из-за этого возникают проблемы:

- исследователь может сделать слишком сильный вывод;
- природоохранная организация может неверно расставить приоритеты;
- чиновник или менеджер может принять решение на слабой основе;
- авторы данных могут не получить обратную связь;
- источники GBIF могут быть процитированы неправильно или неполно.

EcoGenesis Evidence Atlas закрывает этот разрыв между “данные доступны” и “данные можно безопасно использовать для решения”.

## 3. Что такое GBIF простыми словами

GBIF — это международная инфраструктура открытых данных о биоразнообразии. Через GBIF можно получить миллионы записей о том, где и когда находили разные виды: растения, животные, грибы, насекомых и другие организмы.

Но GBIF не обязан сам за пользователя решать:

- достаточно ли этих данных для конкретного вывода;
- можно ли говорить об отсутствии вида;
- какие записи имеют проблемы;
- какие наборы данных надо цитировать;
- где лучше проводить новые наблюдения.

EcoGenesis строится именно как слой интерпретации поверх GBIF.

## 4. Что решает проект

Проект решает задачу ответственного использования данных GBIF.

Вход:

```text
вид или группа видов + регион + цель анализа
```

Например:

```text
Aedes albopictus + Spain + invasive watch
```

Выход:

- короткий decision memo;
- карта доказательств;
- readiness score;
- безопасные и запрещенные claims;
- риски качества данных;
- список источников и цитирований;
- publisher feedback;
- полный evidence pack для проверки.

## 5. Почему это сильнее обычной карты

Обычная карта отвечает: “где есть записи?”

EcoGenesis отвечает:

```text
Что я могу решить на основе этих записей?
Что нельзя утверждать?
Что надо проверить дальше?
Кого и как цитировать?
Какие данные мешают надежному выводу?
```

Это делает инструмент полезным не только для красивой демонстрации, но и для реальной научной, образовательной и управленческой работы.

## 6. Основные пользователи

### Исследователь

Получает быстрый обзор качества данных перед анализом. Видит, какие выводы поддерживаются, а какие требуют дополнительной модели или DOI-backed GBIF download.

### Специалист по охране природы

Может понять, где есть реальные наблюдения, где пробелы, где надо планировать полевые обследования и где нельзя делать вывод об отсутствии вида.

### Data manager или GBIF node

Получает список наборов данных, проблем и готовые шаблоны сообщений для publishers.

### Образовательный пользователь

Может понять GBIF-данные без сложной статистики: карта, memo, claims, warning-и и next action.

### EcoGenesis как приложение

Получает доверительный слой: не просто AI/карта, а проверяемый evidence workflow с API, экспортом, схемой и reproducibility.

## 7. Два режима интерфейса

### Presentation

Короткая витрина для жюри и новых пользователей.

Показывает:

- что решает инструмент;
- live GBIF status;
- текущий verdict;
- карту;
- safe claims;
- blocked claims;
- кнопку перехода в Workbench;
- скачивание Evidence Pack.

### Work with GBIF

Рабочий режим.

Пользователь:

1. ищет таксон через GBIF;
2. выбирает taxonKey;
3. выбирает регион или bbox;
4. выбирает цель;
5. нажимает Generate Evidence Passport;
6. получает результат.

Важно: по умолчанию используется Live GBIF. Fixture-данные скрыты от обычного пользователя и нужны только для тестов.

## 8. Что такое Evidence Passport

Evidence Passport — это пакет, который объясняет состояние данных для одного запроса.

В нем есть:

- `decision_memo.md` — короткий ответ человеку;
- `passport.html` — автономная страница отчета;
- `records.geojson` — записи для карты;
- `quality_metrics.csv` — метрики качества;
- `gap_priorities.csv` — приоритетные клетки для обследования;
- `dataset_contributions.csv` — вклад наборов данных;
- `citations.md` — цитирование и DOI warning;
- `publisher_issue_templates.md` — готовые сообщения data publishers;
- `evidence_pack.json` — машинно-читаемый пакет;
- `evidence_pack.zip` — полный архив.

## 9. Самая важная функция: Claim Guardrails

Claim Guardrails говорит пользователю:

- supported claims: это можно утверждать;
- weak claims: это можно предположить осторожно;
- blocked claims: это нельзя утверждать;
- required verification: это нужно проверить перед публикацией.

Пример blocked claim:

```text
Absence cannot be inferred from empty or low-evidence grid cells.
```

То есть пустая клетка на карте — это не доказательство отсутствия вида. Это цель для дальнейшего обследования.

## 10. Citation Autopilot

GBIF-данные надо правильно цитировать. Если пользователь скачал карту или API-ответ, это еще не всегда достаточно для публикации.

EcoGenesis сохраняет:

- datasetKey;
- вклад каждого dataset;
- license;
- publisher;
- source mode;
- run parameters;
- предупреждение о необходимости DOI-backed GBIF occurrence download или derived dataset.

Это помогает не потерять атрибуцию и заранее видеть, чего не хватает для публикационного использования.

## 11. Publisher Feedback

Если данные имеют проблемы, инструмент не просто ругается. Он формирует обратную связь:

- какой dataset;
- какая проблема;
- сколько записей затронуто;
- какой suggested fix;
- готовый текст сообщения.

Это полезно для GBIF-сети, потому что reuse начинает возвращать качество обратно к источникам.

## 12. Что проект дает GBIF

Для GBIF проект полезен тем, что:

- повышает ответственное использование GBIF-mediated data;
- снижает риск неправильных выводов по occurrence records;
- усиливает citation compliance;
- помогает обнаруживать data gaps;
- создает обратную связь для publishers;
- показывает, как можно строить понятные decision tools поверх открытых данных.

Проект не заменяет GBIF. Он делает GBIF-данные более понятными, проверяемыми и пригодными для решений.

## 13. Что проект дает EcoGenesis

Для EcoGenesis это может стать центральным модулем доверия.

Вместо абстрактной большой платформы EcoGenesis получает конкретный продукт:

```text
GBIF Evidence Passport API + UI + export pack
```

Это можно встроить в:

- карту биоразнообразия;
- AI-агента;
- кабинет исследователя;
- кабинет data manager;
- образовательный режим;
- будущие модули по мониторингу, рискам, инвазивным видам, conservation planning.

Главный эффект: приложение начинает не просто показывать данные, а объяснять надежность данных.

## 14. Почему это конкурсно сильная заявка

Победные конкурсные проекты обычно не обещают “платформу обо всем”. Они показывают конкретный работающий инструмент, который можно проверить.

EcoGenesis Evidence Atlas попадает в эту логику:

- узкий и понятный scope;
- live GBIF API;
- reproducible export;
- open-source;
- понятная польза для GBIF community;
- видео и operating instructions;
- тесты backend/frontend;
- demo cases;
- machine-readable schema.

## 17. Roadmap после конкурса

### Этап 1: конкурсный MVP

Цель: доказать, что Evidence Passport нужен и работает.

### Этап 2: публичный hosted demo

Цель: дать людям запускать tool без локальной установки.

### Этап 3: DOI-backed workflow

Цель: автоматически помогать переходить от API-паспорта к публикационно корректному GBIF download/derived dataset.

### Этап 4: пользовательские проекты

Цель: сохранять наборы запросов, сравнивать регионы и строить серию evidence passports.

### Этап 5: EcoGenesis platform layer

Цель: интегрировать Evidence Passport в более широкую платформу EcoGenesis как модуль доверия для biodiversity intelligence.

## 18. Главная фраза проекта

EcoGenesis Evidence Atlas превращает GBIF-данные из “точек на карте” в проверяемое решение: что можно утверждать, что нельзя, что нужно процитировать и что исправить дальше.
