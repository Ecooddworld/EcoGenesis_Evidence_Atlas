EcoGenesis Evidence Atlas is now the Molecular Evidence Conversion and Repair Engine for GBIF.

The core problem is not drawing another biodiversity map. The problem is deciding which DNA-derived records can safely become GBIF-ready evidence, which records must be downgraded to a safer rank, and which records need repair before publication.

The first working layer is the Barcode-to-GBIF Evidence Compiler. A user can upload Sequence ID, BLAST, BOLD, UNITE or laboratory CSV results, validate the input, generate a run, and inspect every safe, weak, blocked and not-publishable decision.

The compiler does not copy the nearest top-hit species into an export. It checks identity, query coverage, ambiguity, lowest common ancestor, barcode gap evidence, diagnostic k-mers and GBIF/DNA-derived metadata. Species-level output is allowed only when the gates pass.

The output is an Evidence Pack, not a black-box score. It includes decision tables, publication blockers, Darwin Core and DNA-derived templates, methods, citations, graph provenance, VSEA rows, theorem checks and AI guardrail audits.

The Observatory layer adds a bounded GSIG evidence graph. It links GBIF occurrence context, hashed snapshots, molecular segments, claim states, blockers, repairs and exports. The GBIF map is shown as occurrence context only; it never upgrades a weak barcode claim into support.

The graph explorer makes the claim boundary visible. Each node and edge can be inspected as an evidence object, and the Judge view keeps proof obligations, caveats and export status separate.

The competition packs include two frozen 100-sequence runs: a balanced contest batch and an adversarial fail-closed batch. Both preserve expected decision classes, theorem status, graph roundtrip checks and downloadable evidence packs.

EcoGenesis changes the workflow from “top hit equals species” to “what is the safest claim supported by this evidence, and what repair would make more records GBIF-ready?”
